Jitsi Meet mit Docker fuer Synology

Jitsi ist eine Sammlung von Open Source Projekten, die das einfache und sichere Betreiben von Video-Konferenzenz Loesungen ermoeglicht. Jitsi Meet ist ein komplett verschluesseltes, 100%-iges Open Source Video Konfernez System, zur taeglichen freien Nutzung, ohne dass Accounts, oder Lizenzen noetig sind. Das Paket basiert auf einem Repository, mit allen noetigen Tools, um den Jitsi-Meet Stack auf Docker durch Docker Compose zu Betreiben verpackt in Scripts und UI.

Quick Start

Die Basis ist bereits gesetzt. Weitere Anpassungen verlangen jitsi-cli reset zum Anwenden. Woechentliches Jitsi-cli refresh laedt neue Releases. Bei NAT Router mit dynamischer IP setzt jitsi-cli start die DOCKER_HOST_ADDRESS jeden Morgen auf die oeffentliche IP. Nutze Reverse-Proxy zum Routen https-443 auf Container http-8080.

Architecture

Eine Jitsi-Meet Installation besteht aus folgende Komponenten:
  • ein Web Interface (Lande-Seite) & einen XMPP Server (fuer Autentifizierung)
  • eine Konferenz Fokus Komponente & ein Video Router (oder mehrere)
  • ein SIP Gateway fuer Audio Calls & eine Broadcasting Infrastruktur fuer Aufnahmen Konferenz Streaming.

    • Frequently Asked Questions

    Ich habe Probleme mit SSL und Lest-Encrypt; Ich habe es nach Anleitung im Docker Envrionment Aktiviert
    Das integrierte Lets-Encrypt Feature von Jitsi funktioniert auf Synology nicht, da Diese den port 80 / 443 bedient. Es ist aber auch nicht noetig. Statt dessen nutzt man den Synology Reverse Proxy in Systesteuerung, Anwendungsportal: Mapping http(s) auf localhost:8080 (oder konfigurierter Port). Man kann aber auch ein Synology verwaltetes Lets-Encrypt Zertifikat in die Container syncen (Power-User Feature via 'SYNOPKG_SYNO_CERT_SYNC=1').
    I habe alle Features aktiviert und der Jibri Container startet nicht
    Wie bei der Installation hingeweisen ist Jibri experimentell und startet aktuell nicht, solange kein ALSO audio loopback auf dem Host installiert ist (siehe Issue 721).
    Der Jitsi Video Stream bricht nach einigen Sekunden ab. Es hatte bereits funktioniert
    Fuehre jitsicli health aus. Vermutlich hat sich die dynamische oeffenltiche IP geandert und ist nicht in Jitsi aktualisiert. Um das zu erreichen Starte das Paket jeden Morgen.
    Wie kann ich sicherstellen, dass niemand Aussenstehender Jitsi fuer Konferenzen missbraucht?
    Dieses Jitsi Paket nutzt Authetifizierung als Default, daher auch die Moderator und Passwort Abfrage bei Installation. Benutehr koennen hinzugefuegt werden ueber jitsi-cli add-usr, was bei jedem Reset / Refresh Aktualisiert wird.
    Wie kann ich Jitsi Anpassen, z.B. mit Impressum und Datenschutz?
    Hier kommt die Imprint-Fusszeile ins Spiel und das Config-Mod Skript. Siehe Vorlagen, besonders das Mod-Skript hat interesannte Datenschutz Feature vorbereitet.
    Ich kann Das Konfigurations-File nicht Aktualisieren, der Spwichern-Knopf is deaktiviert
    Die JavaScript Klasse erkennt zuweilen kein 'inline-edit'. Das Hinzufuegen eines Leerzeichens am ende der Zeitle aktiviert den Speichern-Knopf.
