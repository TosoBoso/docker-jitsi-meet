Jitsi Meet on Docker for Synology

Jitsi is a set of Open Source projects that allows you to easily build and deploy secure video conferencing solutions. Jitsi Meet is a fully encrypted, 100% Open Source video conferencing solution that you can use all day, every day, for free with no account needed. This package is based on a repository that contains the necessary tools to run a Jitsi-Meet stack on Docker using Docker Compose wrapped in scripts and UI.

Quick Start

Install did set the basics. Further configuration changes require runing jitsi-cli reset to apply it. Run jitsi-cli refresh weekly for new releases. When using a NAT router with dynamic IP schedule jitsi-cli start each morning to set DOCKER_HOST_ADDRESS to public ip. Use Synology Reverse-Proxy to route https-443 to container http-8080.

Architecture

A Jitsi-Meet installation can be broken down into the following components:
  • a web interface (for landing) & a XMPP server (for user authentication)
  • a conference focus component & a video router (could be more than one)
  • a SIP gateway for audio calls & a Broadcasting Infrastructure for recording or streaming a conference.

    • Frequently Asked Questions

    I have trouble with SSL and Lets-Encrypt, I enabled it in Docker Environment as per HowTo
    The embedded Lets-Encrypt feature of Jitsi does not work since Synology hold the port 80 and 4443. However it is not necessary to give SSL certificate to container. Instead use the Synology Reverse Proxy feature under Control Panel, Application Portal mapping http(s) to localhost:8080 (or your respective port). On the other hand the package allows via 'SYNOPKG_SYNO_CERT_SYNC=1' to sync a certificated maintainind via Lets-Encrypt on Synology level into the container, a power user feature.
    I enabled all features and the Jibri container fails to run
    As indicated at install Jibri is experimental and will currently not start in docker unless ALSA audio loopback is installed on host (see Issue 721).
    Jitsi video stream stops after a few seconds. It was working initally
    Run jitsicli health. Probably your public dynamic ip address changed and it is not reflected in Jitsi. To apply this start the package each morning.
    How can I make sure no one anonymous is misusing my Jitsi starting conferences?
    This Jitsi package runs with authentication mode by default, hence the initial moderator user and password at install. You cann add more via jitsi-cli add-usr. At each reset / refresh the users will be added again.
    How can customize Jitsi e.g. adding my imprint and data protection link?
    This is where the Imprint-Footer and Config-Mod script comes into play. Look at the templates. Respectively the mod-script has interesting features prepared to be uncommented.
    I cannot update the configuration file, the save button is disabled
    The JavaScript class does not detect inline editing in some cases. Add space at end of new line to get the save button enabled.
